What is attention?

Attention allows models to focus on relevant parts of the input by computing relationships between elements using queries (Q), keys (K), and values (V).

Core idea: similarity(Q, K) → weights → weighted sum of V

Types of attention

Soft vs Hard

  • Soft: differentiable, standard in Transformers
  • Hard: discrete selection, harder to train

Global vs Local

  • Global: full attention (O(n²))
  • Local: windowed attention (O(nw))

Self / Cross / Multi-head

  • Self-attention: same sequence
  • Cross-attention: different sequences
  • Multi-head: parallel attention subspaces

Efficient attention

  • Sparse attention (BigBird, Longformer)
  • Low-rank (Linformer)
  • Linear attention (Performer)
  • FlashAttention (IO-aware kernels)
  • Streaming / chunked attention
  • Quantized attention (FP8 / INT4)

Recent advances (2024–2026)

FlashAttention evolution

  • FlashAttention-2/3 improves tiling and parallelism
  • Eliminates large memory overhead

Low precision attention

  • FP8 → FP4 → INT4
  • SageAttention improves quantization stability

Long-context models

  • 100K → 1M+ tokens
  • KV-cache optimization

Hybrid architectures

  • Attention + State-space models (Mamba)
  • Linear-time alternatives emerging

MoE + attention

  • Sparse expert routing
  • Trillion-parameter scaling
Modern progress focuses on efficiency and hardware co-design rather than changing the core attention formula.

Hardware implementations

GPUs / TPUs

  • Matrix multiplication optimized
  • FlashAttention kernels

AI accelerators

  • Cerebras (wafer-scale)
  • Graphcore, Groq

Trends

  • Low precision (FP4, INT4)
  • KV-cache optimized inference
  • Memory bandwidth optimization
